# Logue(로그) 프로젝트 청사진

## 1. 서비스 개요 (PRD 기반)

- **프로젝트명:** **Logue (로그)**
- **한 줄 정의:** 매일의 기록을 통해 원어민의 감각을 체득하는 AI 영어 다이어리 서비스
- **핵심 가치:**
    1. **Consistency (지속성):** 부담 없는 기록 습관 형성
    2. **Growth (성장):** 내 기록이 학습 데이터가 되는 경험
    3. **Refinement (정교함):** AI를 통한 자연스러운 문장 교정
- **타겟 고객:**
    - **2040 자기계발층:** 업무나 일상에서 더 세련된 영어를 구사하고 싶은 직장인 및 학생
    - **습관 형성형 학습자:** 강의 시청보다 본인의 이야기를 기록하며 배우는 것을 선호하는 사용자

---

## 2. 핵심 기능 구현 계획

### 1단계: [인트로] 매력적인 첫인상 (완료)
- **목표:** 서비스의 철학을 담은 소개 페이지로 사용자의 학습 동기 부여.
- **구현 내용:** Hero 섹션 문구 및 디자인 재구성, 기능 소개 시각화.

### 2단계: [인증] Google 계정으로 시작하기 (완료)
- **목표:** Firebase Authentication을 연동하여 간편 로그인 기능 구현.
- **구현 내용:** Firebase 연동, `AuthContext`를 통한 전역 상태 관리, 헤더 UI에 로그인/로그아웃 기능 통합.

### 3단계: [경로 보호] 나만의 다이어리 공간 (완료)
- **목표:** 로그인한 사용자만 접근할 수 있는 `/diary` 페이지 구현.
- **구현 내용:** `useAuth` 훅을 사용한 인증 상태 확인, 비로그인 사용자 리디렉션, 로딩 UI를 통한 UX 향상.

### 4단계: [AI 코칭] 핵심 가치 구현 (완료)
- **목표:** 사용자의 영어 일기를 분석하고, 실력 향상을 위한 정교한 피드백을 제공하는 AI 코칭 시스템 구축.

- **4-1. [UI/UX] 글쓰기 경험 업그레이드 (완료):**
    - `react-quill` 라이브러리를 설치하여 Rich Text Editor 구현.
    - AI 피드백을 보여줄 `AIAnalysisResult.tsx` 컴포넌트 레이아웃 구성.

- **4-2. [AI] Gemini 코칭 엔진 연동 (완료):**
    - `@google/generative-ai` SDK 설치 및 Server Action `analyzeDiaryEntry` 함수 구현.
    - 원어민 튜터 역할을 수행하는 정교한 AI 프롬프트 설계 (문장별 분석, 대안, 핵심 표현 설명 포함 JSON 반환).

- **4-3. [DB] 학습 기록 저장 및 개인화 (완료):**
    - Firestore 데이터 모델을 `users/{uid}/entries` 와 `users/{uid}/vault` 구조로 설계.
    - 일기 제출 시, 원본 및 AI 분석 결과를 Firestore에 저장.
    - '보관함에 추가' 기능 구현.

- **4-4. [자산화] 학습 히스토리 조회 (대기):**
    - `/diary/history` 경로에 작성 일기 목록 페이지 생성.
    - 과거 일기 및 AI 피드백 상세 조회 기능 구현.

## 5. 5단계: [배포] 전 세계에 Logue 공개

- **목표:** Firebase App Hosting을 사용하여 Logue 서비스를 실제 프로덕션 환경에 배포하고, 전 세계 사용자가 접근할 수 있는 공개 URL을 확보.
- **5-1. [인프라] Firebase 프로젝트 요금제 업그레이드 (완료):**
    - App Hosting 기능을 사용하기 위해 Firebase 프로젝트를 'Blaze' 요금제로 성공적으로 업그레이드 완료.
- **5-2. [백엔드] App Hosting 백엔드 생성 (완료):**
    - `firebase apphosting:backends:create` 명령어를 실행하여 App Hosting 인프라 프로비저닝 및 GitHub 리포지토리 연결 성공.
- **5-3. [설정] 배포 설정 수정 (완료):**
    - 잘못 입력된 루트 디렉토리(`y`)를 올바른 값(`/`)으로 Firebase 콘솔에서 직접 수정 완료.
- **5-4. [CI/CD] 최종 배포 실행 (진행중):**
    - 수정된 설정을 기반으로 새로운 배포(Rollout)를 시작하여 GitHub 리포지토리의 코드를 실제 프로덕션 환경에 빌드 및 배포.
- **5-5. [출시] 서비스 URL 확인 (대기):**
    - 배포가 완료된 Logue 서비스의 라이브 URL을 확인하고 접속 테스트.
